## Flow 1: First Launch (Установка → Онбординг → Paywall → Home)

**Цель:** Провести нового пользователя от установки до aha-момента (первый скан) и показать paywall на пике эмоции.

### Happy Path

```
[App Store Install] → Splash Screen (1 сек)
    → Welcome Screen (ценностное предложение + social proof)
    → [Нажимает "Начать"]
    → Onboarding Quiz:
        → Goal Selection ("Что важнее?")
        → Family Size ("Для кого?")
        → Product Preferences ("Что покупаете?")
        → Waste Assessment ("Как часто выбрасываете?")
        → Labor Illusion ("Создаём план..." — 3 сек анимация)
    → Demo Scan (aha-момент):
        → Camera Permission Request (pre-permission экран)
        → [Разрешает камеру]
        → Камера → Фото продукта → Анализ (2–3 сек)
        → Scan Result (SAFE/CAUTION/DANGER) — aha!
    → Paywall (персонализированный, на пике эмоции)
        → [Выбирает "7 дней бесплатно" / "Не сейчас"]
    → Push Pre-Permission ("Узнавать, когда продукты истекают?")
        → [Разрешает / Не сейчас]
    → Home Screen (с результатом первого скана)
```

**Метрики:**
- Onboarding completion: целевой 65% (PRACTICES-BRIEF.md)
- Paywall shown: 100% завершивших онбординг
- Trial start: целевой 15% (PRACTICES-BRIEF.md)
- Push opt-in: целевой 55% (PRACTICES-BRIEF.md)

### Error States

| Ситуация | Экран | Действие |
|----------|-------|----------|
| Нет интернета при первом запуске | No Internet Screen | "Для первого запуска нужен интернет. Подключитесь к Wi-Fi." + кнопка "Повторить" |
| Камера не разрешена | Fallback: демо-видео скана вместо живого | Показать анимированный демо-результат + "Разрешите камеру, чтобы сканировать свои продукты" |
| AI-сервер недоступен (demo scan) | Fallback: предзаписанный результат | Показать mock-результат демо-скана (курица → CAUTION), чтобы не потерять aha-момент |
| Paywall не загружается (Adapty) | Пропустить paywall | Перейти к Home, показать paywall позже при первом лимит-триггере |
| Пользователь закрывает приложение на quiz | Сохранить прогресс quiz | При следующем открытии — продолжить с того же экрана, не начинать заново |

### Edge Cases

| Ситуация | Поведение |
|----------|-----------|
| Переустановка (returning user) | Если есть аккаунт — предложить войти (skip quiz). Если нет — полный онбординг |
| Пользователь нажимает "Skip" на всех экранах | Пропустить quiz → показать demo scan → paywall. Минимальный путь: Welcome → Demo → Paywall → Home |
| Очень старое устройство (iPhone SE 1st gen) | Упрощённые анимации, без Labor Illusion (прямой переход) |
| Пользователь фотографирует не еду (demo scan) | Результат: "Мы не смогли распознать продукт. Попробуйте сфотографировать еду ближе" |

## Flow 2: Freshness Scan (Core Action)

**Цель:** Сфотографировать продукт → получить оценку свежести за <3 секунды → принять решение.

### Happy Path

```
[Пользователь на Home Screen]
    → [Нажимает кнопку камеры (центральная, крупная)]
    → Camera Screen (камера открывается за <1 сек)
        → [Наводит на продукт, делает фото]
    → Scan Processing (анимация 2–3 сек, "Анализируем...")
    → Scan Result Screen:
        → Индикатор: SAFE (зелёный) / CAUTION (жёлтый) / DANGER (красный)
        → Confidence: "87% уверенность"
        → Что обнаружено: "Курица выглядит свежей: равномерный розовый цвет, нет потемнений"
        → Совет хранения: "Используйте в течение 1–2 дней или заморозьте"
        → Disclaimer: "Визуальная оценка. Не обнаруживает бактерии."
        → [Кнопки: "Добавить в My Fridge" / "Сканировать ещё" / "Поделиться"]
        → Feedback: "Этот результат полезен? 👍 👎"
    → [Нажимает "Добавить в My Fridge"]
    → Quick Add (имя продукта предзаполнено из скана)
        → [Подтверждает]
    → My Fridge (продукт добавлен, toast "Курица добавлена")
```

**Метрики:**
- Время до результата: <3 сек (F1 acceptance criteria)
- Scans/user/week: целевой 3 → 7 (PRODUCT-VISION.md)
- Feedback rate: целевой 10%+

### Error States

| Ситуация | Экран | Действие |
|----------|-------|----------|
| Камера не разрешена | Permission Screen | "FreshCheck нужна камера для сканирования. Разрешите в настройках." + кнопка "Открыть настройки" |
| Нет интернета | Offline Warning | "Для анализа нужен интернет. Попробуйте позже." + "Открыть Storage Guide (работает offline)" |
| AI timeout (>10 сек) | Retry Screen | "Анализ занимает дольше обычного." + кнопка "Повторить" / "Отмена" |
| AI не распознал продукт | Result: Unknown | "Не удалось определить продукт. Попробуйте: 1) ближе, 2) лучше освещение, 3) один продукт за раз" |
| Фото слишком тёмное/размытое | Camera Hint | "Фото получилось нечётким. Попробуйте ещё раз при лучшем освещении." (до отправки на анализ) |
| Low confidence (<70%) | Result: CAUTION | Default to CAUTION: "Мы не уверены на 100%. На всякий случай — осторожность." |
| Raw poultry detected | Result: специальный | НИКОГДА "SAFE". Всегда: "Готовьте до 74°C (165°F) независимо от внешнего вида." |

### Edge Cases

| Ситуация | Поведение |
|----------|-----------|
| Free user: 5-й скан за день | Скан выполняется. После результата — soft banner: "Остался 1 скан на сегодня. Plus = безлимитно" |
| Free user: 6-й скан (лимит) | Вместо камеры — Paywall: "Дневной лимит исчерпан. Разблокируйте безлимитные сканы." |
| Сканирование не-еды (рука, стол) | "Мы не нашли продукт на фото. Наведите камеру на еду." |
| Сканирование упаковки (не содержимое) | "Для точной оценки сфотографируйте сам продукт, а не упаковку." |
| Множественные продукты на фото | Определить один, наиболее заметный. "На фото несколько продуктов. Результат для: [название]. Сканируйте остальные по одному." |

## Flow 3: Return Visit (Повторное открытие)

**Цель:** Показать ценность при каждом возвращении — что изменилось в холодильнике, что нужно использовать.

### Happy Path — Открытие через push

```
[Push: "🍗 Курица — используйте сегодня или заморозьте"]
    → [Нажимает на push]
    → Deep link → My Fridge, карточка "Курица" раскрыта
        → [Видит: "Последний день. Советы: приготовить / заморозить"]
        → [Нажимает "Рецепты с курицей"]
    → Recipes (фильтр: курица, <30 мин)
        → [Выбирает рецепт]
    → Recipe Detail
        → [Готовит]
    → [Свайп "Использовал" на карточке курицы в My Fridge]
    → My Fridge (курица удалена, "+$8.50 сэкономлено" toast)
```

### Happy Path — Органическое открытие

```
[Иконка приложения]
    → Home Screen:
        → Summary: "3 продукта скоро истекают"
        → Quick Scan CTA (центральная кнопка)
        → Последний скан: "Молоко — SAFE (вчера)"
    → [Нажимает "3 продукта скоро истекают"]
    → My Fridge (отсортировано по срокам)
        → [Видит жёлтую/красную карточку]
        → [Сканирует продукт или смотрит рецепты]
```

### Happy Path — Утреннее открытие (push-дайджест)

```
[Push утром: "3 продукта скоро истекают — проверьте"]
    → [Нажимает]
    → My Fridge (overview: 3 жёлтых/красных карточки сверху)
    → [Просматривает, решает: использовать / заморозить / выбросить]
```

### Error States

| Ситуация | Поведение |
|----------|-----------|
| Push пришёл, но продукт уже удалён | Deep link → My Fridge. Toast: "Этот продукт уже отмечен." |
| Нет интернета при открытии | Показать My Fridge из кэша (offline). Scan недоступен. Badge: "Offline — данные холодильника актуальны." |
| Сессия истекла (long absence) | Показать Home. Если 30+ дней неактивности — мягкий re-onboarding: "С возвращением! Вот что нового." |

### Edge Cases

| Ситуация | Поведение |
|----------|-----------|
| My Fridge пуст (новый пользователь / всё использовано) | Empty state: "Холодильник пуст! Добавьте продукты после покупки." + CTA "Добавить" + "Сканировать" |
| Все продукты в зелёной зоне | Home: "Всё свежее! 0 продуктов истекают на этой неделе 🎉" |
| Free user, push не разрешены | Вместо push — in-app banner при открытии: "У вас 2 продукта, которые скоро истекут" |
| Пользователь не открывал 7+ дней | Win-back push (если разрешены): "Проверьте холодильник — возможно, что-то пора использовать" |

## Flow 4: Upgrade Flow (Free → Plus)

**Цель:** Показать ценность Plus в момент, когда пользователь упирается в лимит или видит premium-фичу.

### Happy Path — Soft Limit

```
[Free user делает 5-й скан за день]
    → Scan Result (нормальный)
    → Soft Banner внизу результата:
        → "Последний бесплатный скан на сегодня. Plus = безлимитно."
        → [Нажимает "Подробнее"]
    → Paywall:
        → Персонализированный заголовок: "Защитите семью каждый день"
        → Benefits: безлимитные сканы, push-уведомления, рецепты, расширенный холодильник
        → Планы: $4.99/мес | $39.99/год (выделен, "Экономия 33%") | 7 дней бесплатно
        → Social proof: "4.5 ★ от 12,400 семей"
        → Restore purchases
    → [Нажимает "Попробовать 7 дней бесплатно"]
    → Apple/Google Pay подтверждение
    → Success Screen:
        → "Добро пожаловать в Plus! 🎉"
        → "Вот что теперь доступно:" — список разблокированного
        → CTA: "Начать сканирование"
    → Home (Plus badge в профиле)
```

### Happy Path — Hard Limit

```
[Free user пытается сделать 6-й скан]
    → [Вместо камеры] → Paywall:
        → "Дневной лимит: 5 сканов. Разблокируйте безлимитные сканы с Plus."
        → [Те же варианты подписки]
    → [Выбирает план → покупка → Success → Home]
```

### Happy Path — Premium Feature Trigger

```
[Free user нажимает на вкладку "Recipes"]
    → Recipes Screen с blur/lock overlay:
        → Превью 2–3 рецептов (размытые)
        → "Рецепты из ваших продуктов — только в Plus"
        → CTA: "Попробовать бесплатно"
    → Paywall → [покупка] → Recipes (разблокированы)
```

### Error States

| Ситуация | Поведение |
|----------|-----------|
| Ошибка оплаты (карта отклонена) | "Не удалось обработать оплату. Проверьте данные карты." + "Попробовать снова" / "Другой способ" |
| Adapty недоступен | "Сервис подписок временно недоступен. Попробуйте позже." + продолжить как Free |
| Пользователь отменяет на Apple Pay | Вернуть на Paywall. Не закрывать, не навязывать |
| Restore purchases не находит покупку | "Подписка не найдена для этого Apple ID. Использовали другой аккаунт?" |

### Edge Cases

| Ситуация | Поведение |
|----------|-----------|
| Trial уже использован | Paywall без trial: прямая покупка $4.99/мес или $39.99/год |
| Пользователь подписался и отменил | Grace period: доступ до конца оплаченного периода. За 3 дня — push: "Подписка скоро заканчивается" |
| Family tier (будущее) | Если >3 в семье — предложить Family ($7.99/мес) вместо Plus |
| Plus user уже подписан, видит paywall | Не показывать paywall. Все фичи доступны без ограничений |

## Flow 5: Share Flow (Результат → Шеринг)

**Цель:** Превратить каждый скан в потенциальный вирусный контент (TikTok/Instagram формат).

### Happy Path

```
[Scan Result Screen]
    → [Нажимает "Поделиться"]
    → Share Card Preview:
        → Генерируется карточка:
            → Фото продукта
            → Крупный индикатор: SAFE / CAUTION / DANGER
            → "Проверено FreshCheck AI"
            → QR-код / ссылка на приложение
    → [Нажимает "Поделиться"]
    → iOS/Android Share Sheet:
        → Instagram Stories / WhatsApp / iMessage / Facebook / Copy Link
    → [Выбирает Instagram Stories]
    → Instagram открывается с карточкой
    → [Публикует]
```

### Happy Path — Weekly Report Share

```
[Weekly Report Screen (воскресенье)]
    → "На этой неделе вы сэкономили $23.50"
    → Статистика: 5 использовано, 1 заморожено, 0 выброшено
    → [Нажимает "Поделиться достижением"]
    → Share Card:
        → "Я сэкономил(а) $23.50 на еде за неделю с FreshCheck!"
        → Streak: 14 дней без выброшенной еды
        → CTA: "Попробуйте бесплатно"
    → Share Sheet → [выбор платформы] → публикация
```

### Error States

| Ситуация | Поведение |
|----------|-----------|
| Share Sheet не открывается | Fallback: "Скопировать ссылку" в буфер обмена. Toast: "Ссылка скопирована!" |
| Генерация карточки не удалась | Поделиться текстом: "Я проверил(а) курицу в FreshCheck — SAFE! 🟢 Скачай: [ссылка]" |
| Нет установленных соцсетей | Стандартный Share Sheet покажет доступные опции (iMessage, Mail, Copy) |

### Edge Cases

| Ситуация | Поведение |
|----------|-----------|
| Результат DANGER — стоит ли шерить? | Карточка генерируется для любого результата. DANGER тоже вирусный: "FreshCheck спас меня от этого! 🔴" |
| Free user шерит | Шеринг доступен всем (это виральный канал, не premium-фича) |
| Weekly Report: 0 экономии | Не предлагать шеринг. Показать: "Добавьте продукты, чтобы видеть экономию" |
| Шеринг CAUTION с low confidence | Не показывать confidence score на share card (избежать путаницы). Только SAFE/CAUTION/DANGER |
