## Детали каждого экрана

### 1. Onboarding

#### 1.1 Welcome Screen

| Поле | Описание |
|------|----------|
| **Назначение** | Первое впечатление — передать ценность и вызвать доверие |
| **Откуда** | Splash Screen (первый запуск) |
| **Куда** | 1.2 Goal Selection (или skip → 1.7 Demo Scan) |
| **Ключевые элементы** | Иллюстрация (сканирование продукта), заголовок ("Свежее или нет? Узнайте за 3 секунды"), social proof (рейтинг + кол-во пользователей), CTA "Начать", прогресс-бар (шаг 1/7) |
| **Фича** | F6: Onboarding |

#### 1.2 Goal Selection

| Поле | Описание |
|------|----------|
| **Назначение** | Узнать главную мотивацию для персонализации |
| **Откуда** | 1.1 Welcome |
| **Куда** | 1.3 Family Size |
| **Ключевые элементы** | Вопрос: "Что для вас важнее всего?", 4 варианта (иконка + текст): Безопасность семьи / Меньше выбросов / Экономия / Рецепты из остатков, прогресс-бар (шаг 2/7) |
| **Фича** | F6: Onboarding |

#### 1.3 Family Size

| Поле | Описание |
|------|----------|
| **Назначение** | Узнать размер семьи для расчёта потерь и персонализации push |
| **Откуда** | 1.2 Goal Selection |
| **Куда** | 1.4 Product Preferences |
| **Ключевые элементы** | Вопрос: "Для кого вы проверяете еду?", 4 варианта: Только я / Пара / Семья с детьми / Большая семья (5+), прогресс-бар (шаг 3/7) |
| **Фича** | F6: Onboarding |

#### 1.4 Product Preferences

| Поле | Описание |
|------|----------|
| **Назначение** | Понять, какие продукты покупает, для настройки категорий |

| **Куда** | 1.5 Waste Assessment |
| **Ключевые элементы** | Вопрос: "Какие продукты покупаете чаще?", multi-select chips: Мясо/Птица, Рыба/Морепродукты, Молочные, Фрукты, Овощи, Хлеб/Выпечка, Готовые блюда, Заморозка, прогресс-бар (шаг 4/7) |
| **Фича** | F6: Onboarding |

#### 1.5 Waste Assessment

| Поле | Описание |
|------|----------|
| **Назначение** | Рассчитать персональную экономию — ROI подписки |

| **Куда** | 1.6 Labor Illusion |
| **Ключевые элементы** | Вопрос: "Как часто выбрасываете еду?", slider или 4 варианта: Каждый день / 2–3 раза в неделю / Раз в неделю / Редко, прогресс-бар (шаг 5/7) |
| **Фича** | F6: Onboarding |

#### 1.6 Labor Illusion

| Поле | Описание |
|------|----------|
| **Назначение** | Создать ощущение ценности через "обработку" ответов |
| **Откуда** | 1.5 Waste Assessment |
| **Куда** | 1.7 Demo Scan Result |
| **Ключевые элементы** | Анимация 3–5 сек: "Создаём ваш план...", чекбоксы появляются по одному: ✓ Настраиваем категории / ✓ Рассчитываем экономию / ✓ Подбираем рекомендации, персонализированный итог: "Ваша семья теряет ~$X/год на выброшенной еде" |
| **Фича** | F6: Onboarding, модель Flo (PRACTICES-BRIEF.md) |

#### 1.7 Demo Scan Result

| Поле | Описание |
|------|----------|
| **Назначение** | Aha-момент — показать ценность до paywall |
| **Откуда** | 1.6 Labor Illusion (или Camera с демо-фото) |
| **Куда** | 1.8 Paywall |
| **Ключевые элементы** | Демо-результат сканирования (предзаписанный или живой скан), SAFE/CAUTION/DANGER крупным, описание что обнаружено, совет хранения, CTA: "Хочу сканировать свои продукты!" → Paywall |
| **Фича** | F6: Onboarding, F1: Freshness Scanner |

#### 1.8 Paywall (первый показ)

| Поле | Описание |
|------|----------|
| **Назначение** | Конвертировать на пике эмоции после aha-момента |
| **Откуда** | 1.7 Demo Scan Result |
| **Куда** | Push Pre-Permission (3.2) → Home (2.1) |
| **Ключевые элементы** | Персонализированный заголовок (из quiz: "Защитите семью" / "Экономьте $X/год"), 3–4 benefit-пункта (не фичи!), тарифы: месячный / годовой (выделен, "Экономия 33%") / 7 дней бесплатно, social proof: "4.5 ★ · 12,400 семей", кнопка CTA, ссылки: Restore / Terms / Privacy, кнопка "Не сейчас" (мелким) |
| **Фича** | F8: Paywall |

|------|----------|
| **Назначение** | Главный экран — быстрый доступ к скану + обзор холодильника |

### 3. Modals / Overlays

#### 3.1 Paywall (повторный показ)

| Поле | Описание |
|------|----------|
| **Назначение** | Конвертировать при лимит-триггере |
| **Откуда** | Scan Limit / Fridge Limit / Recipes Tab (free) / Upgrade CTA |

| **Фича** | F8: Paywall |

#### 3.2 Push Pre-Permission

| Поле | Описание |
|------|----------|
| **Назначение** | Объяснить ценность push перед системным запросом |
