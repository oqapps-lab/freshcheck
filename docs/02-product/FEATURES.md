## MVP (Must Have) — Запуск

### F1: Freshness Scanner (AI фото-оценка свежести)

| Поле | Описание |
|------|----------|
| **Название** | Freshness Scanner |
| **User Story** | Как занятая мама, я хочу сфотографировать продукт из холодильника и мгновенно узнать, безопасен ли он, чтобы не рисковать здоровьем детей и не выбрасывать хорошую еду |
| **Acceptance Criteria** | - [ ] Камера открывается за <1 сек с главного экрана |
|  | - [ ] AI-модель распознаёт 15–20 категорий продуктов (курица, говядина, свинина, рыба, молоко, яйца, хлеб, клубника, салат, помидоры, авокадо, бананы, яблоки, сыр, йогурт) |
|  | - [ ] Результат за <3 секунды: SAFE (зелёный) / CAUTION (жёлтый) / DANGER (красный) |
|  | - [ ] Confidence score отображается (например, "87% уверенность") |
|  | - [ ] Текстовое объяснение: что обнаружено, совет по хранению, ссылка на USDA |
|  | - [ ] Disclaimer постоянно видимый: "Визуальная оценка. Не обнаруживает бактерии." |
|  | - [ ] Raw poultry → НИКОГДА "SAFE". Всегда: "Cook to 165°F regardless of appearance" |
|  | - [ ] Low confidence (<70%) → default to CAUTION |
|  | - [ ] Feedback: "Этот результат был полезен? Да/Нет" |
|  | - [ ] Free tier: 5 сканов/день |
| **Экран** | Scan Screen (главный экран, центральная кнопка камеры) → Result Screen |
| **Приоритет** | P0 — блокер запуска (core value proposition) |
| **Сложность** | XL — AI-модель, камера, inference, UI результата |

---

### F2: My Fridge — Трекер продуктов

| Поле | Описание |
|------|----------|
| **Название** | My Fridge |
| **User Story** | Как пользователь, я хочу отслеживать продукты в холодильнике с обратным отсчётом до истечения срока, чтобы ничего не забыть и не выбросить |
| **Acceptance Criteria** | - [ ] Добавление продукта: ручной ввод (имя + категория + дата покупки) |
|  | - [ ] Добавление по штрихкоду: сканирование → авто-заполнение имени + shelf life |
|  | - [ ] Shelf life из USDA FoodKeeper database (400+ продуктов) |
|  | - [ ] Обратный отсчёт: визуальная шкала (зелёный → жёлтый → красный) |
|  | - [ ] Сортировка: "скоро истекает" первым |
|  | - [ ] Категории хранения: холодильник / морозилка / кладовая |
|  | - [ ] Свайп для удаления ("использовал" / "выбросил") |
|  | - [ ] Free tier: до 10 продуктов. Plus: unlimited |
| **Экран** | My Fridge Tab (нижняя навигация) |
| **Приоритет** | P0 — блокер запуска (core retention mechanism) |
| **Сложность** | L — CRUD, база данных, штрихкод-сканер, USDA-интеграция |

---

### F3: Push-уведомления о сроках

| Поле | Описание |
|------|----------|
| **Название** | Expiry Alerts |
| **User Story** | Как занятая мама, я хочу получать уведомления, когда продукт скоро испортится, чтобы успеть его использовать или заморозить |
| **Acceptance Criteria** | - [ ] Push за 1 день до истечения: "Курица — используйте сегодня или заморозьте" |
|  | - [ ] Push в день истечения: "Клубника — последний день. Рецепт смузи?" |
|  | - [ ] Push утром с обзором: "3 продукта скоро истекают" (если есть) |
|  | - [ ] Максимум 3 push в день (настраиваемо) |
|  | - [ ] Время уведомлений: по умолчанию 17:00 (настраиваемо) |
|  | - [ ] Deep link из push → продукт в My Fridge или рецепт |
|  | - [ ] Только для Plus-подписчиков (free = in-app alerts без push) |
| **Экран** | Системные push-уведомления → deep link в My Fridge / Recipe |
| **Приоритет** | P0 — блокер запуска (без push нет retention — RESEARCH-BRIEF.md, Risk 3) |
| **Сложность** | M — push infrastructure, scheduling, preferences |

---

### F4: USDA Storage Guidelines

| Поле | Описание |
|------|----------|
| **Название** | Storage Guide |
| **User Story** | Как пользователь, я хочу быстро узнать, сколько хранится конкретный продукт, чтобы не гадать по датам на упаковке |
| **Acceptance Criteria** | - [ ] Поиск по 400+ продуктам (USDA FoodKeeper database) |
|  | - [ ] Для каждого продукта: срок в кладовой / холодильнике / морозилке |
|  | - [ ] До вскрытия / после вскрытия — раздельные сроки |
|  | - [ ] Tips по хранению: "Храните помидоры на столе, не в холодильнике" |
|  | - [ ] Источник: "По данным USDA" — на каждой карточке |
|  | - [ ] Объяснение дат: sell by / best by / use by — что значат |
|  | - [ ] Поиск работает offline |
|  | - [ ] Кнопка "Добавить в My Fridge" из карточки продукта |
| **Экран** | Guide Tab (нижняя навигация) → Product Detail Screen |
| **Приоритет** | P1 — нужно к запуску (дополняет AI-скан авторитетными данными) |
| **Сложность** | M — поиск, база данных, карточки, offline cache |

### F5: Recipe Engine — "Что приготовить?"

| Поле | Описание |
|------|----------|
| **Название** | What to Cook |
| **User Story** | Как занятая мама, я хочу получить рецепт из продуктов, которые скоро испортятся, чтобы не выбрасывать еду и не ломать голову над ужином |
| **Acceptance Criteria** | - [ ] Рецепты приоритизированы по срокам: "Используйте курицу (день 3), перец (день 4), рис" |
|  | - [ ] Связь с My Fridge: автоматически предлагает рецепты из того, что скоро истекает |
|  | - [ ] Фильтры: время готовки (15/30/60 мин), сложность, диета (vegetarian, gluten-free) |
|  | - [ ] Каждый рецепт: ингредиенты, шаги, время, порции |
|  | - [ ] AI-генерация рецептов через OpenAI API |
|  | - [ ] "Недостающие ингредиенты" — список того, чего нет в My Fridge |
|  | - [ ] Только для Plus-подписчиков |
| **Экран** | Recipes Tab (нижняя навигация) → Recipe Detail Screen |
| **Приоритет** | P1 — нужно к запуску (retention + daily engagement) |
| **Сложность** | L — AI-интеграция, фильтры, связь с My Fridge, UI рецептов |

### F6: Onboarding

| Поле | Описание |
|------|----------|
| **Название** | First Scan Onboarding |
| **User Story** | Как новый пользователь, я хочу за 30 секунд понять, как работает приложение, и сделать первый скан, чтобы увидеть ценность до регистрации |
| **Acceptance Criteria** | - [ ] 3 экрана: "Сфотографируй" → "Узнай за 3 сек" → "Отслеживай свой холодильник" |
|  | - [ ] Первый скан — БЕЗ регистрации (value before signup) |
|  | - [ ] После первого скана → предложение создать аккаунт |
|  | - [ ] Skip-кнопка на каждом экране |
|  | - [ ] Анимация демо-скана (если камера ещё не разрешена) |
| **Экран** | Onboarding Flow (3 экрана) → Camera → Result → Signup |
| **Приоритет** | P1 — нужно к запуску (без onboarding — потеря пользователей) |
| **Сложность** | S — 3 экрана, простая логика |

## Won't Have (сейчас) — и почему

| Фича | Почему НЕ делаем |
|------|-----------------|
| **Meal planning** | Отдельная вертикаль с высокой сложностью. Конкурирует с Mealime, Paprika. Не core value |
| **Grocery list** | Тривиальная фича, которую делают 100 приложений. Не дифференцирует |
| **Calorie tracking** | Другой use case (nutrition vs safety). MyFitnessPal доминирует. Не наша ниша |
| **Smart fridge интеграция** | Samsung/LG API ненадёжны, малый % пользователей. Слишком ранний рынок |
| **Social feed / community** | Cold start problem. Вертикальное сообщество трудно построить. Reddit уже выполняет эту роль |
| **B2B (FreshCheck Pro)** | Другой продукт, другие requirements. Year 2+ после потребительского PMF |
| **Internationalization** | Фокус на US market first. i18n добавляет complexity. Year 2+ |
| **Voice assistant** | "Hey Siri, is my chicken still good?" — интересно, но малый impact на MVP |
| **AR overlay** | Наведи камеру на холодильник и увидь сроки каждого продукта — cool, но технически сложно и не validated |
