# v1.1 — Incremental Restore of Deferred Vendors

> **STATUS: ✅ COMPLETE 2026-05-09.** All 4 phases shipped to TestFlight in
> sequence (build #15 ATT, #16 Adapty plugin, #17 FCM messaging, #18 AppsFlyer).
> Two-layer fix recipe + TARGET_DIRS extension worked on every vendor.



> Started after 2026-05-09 ship of v0.2.0 build #14 (sans AppsFlyer / FCM / ATT / Adapty plugin).
> Strategy: re-add ONE vendor per build cycle. Verify each builds + works in TestFlight before
> proceeding. Don't batch; the whole point is to catch the breaker if it returns.

## Rule of thumb

Every restore = 1 separate EAS build + TestFlight verify. Each phase is a single PR-sized commit.
If a phase fails: roll back the commit, file an issue against the SDK, move to next.

## Phase order (least → most risky)

| # | Vendor | Risk | Why this order |
|---|---|---|---|
| 1 | `expo-tracking-transparency` | LOW | Expo-native package, no header drama. Was prophylactically dropped because it was bundled with AppsFlyer; adding back is cosmetic. |
| 2 | `react-native-adapty` plugin | LOW | Just writes Android backup config. iOS build path unaffected. SDK already works via JS. |
| 3 | `@react-native-firebase/messaging` | MEDIUM | Has its own RCT-import bug. `scripts/patch-rnfb-headers.js` walks all RNFB/* dirs, should auto-patch. Verify regex coverage. |
| 4 | `react-native-appsflyer` | HIGH | Same RCT-import bug class as messaging. Need to extend patch script to walk `react-native-appsflyer/` AND adjust regex. Or pivot to Adjust. |

## Phase 1 — Restore `expo-tracking-transparency`

```bash
npm install --save-exact --legacy-peer-deps expo-tracking-transparency@~55.0.13
```

`app.json` plugins (insert just after `expo-camera`, before `expo-notifications`):
```json
[
  "expo-tracking-transparency",
  {
    "userTrackingPermission": "FreshCheck uses your device identifier to measure ad performance and improve our recommendations."
  }
]
```

Then: build production. Expected: PASS (no native code change beyond Info.plist key, which we already had via `NSUserTrackingUsageDescription` in app.json).

**Code already wired:** `src/lib/appsflyer.ts:initAppsFlyerWithATT()` already calls `requestTrackingPermissionsAsync()` via lazy `require()`. With ATT package present, the call resolves; without it, no-op. So no JS change needed.

**Why this works without AppsFlyer:** ATT prompt is just a system dialog. Without AppsFlyer, the user's "Allowed/Denied" answer is captured by the OS, and any future SDK that reads IDFA respects it. We can ship the prompt now, attribution later.

## Phase 4 — Restore AppsFlyer (or pivot to Adjust)

### Option A — restore AppsFlyer

```bash
npm install --save-exact --legacy-peer-deps react-native-appsflyer@^6.17.9
```

`app.json` plugins (after Adapty):
```json
[
  "react-native-appsflyer",
  {
    "shouldUseStrictMode": true
  }
]
```

**Critical**: AppsFlyer SDK has the same `RCTPromiseRejectBlock` non-modular import in
its `.h` files as RNFB messaging. Extend `scripts/patch-rnfb-headers.js` to also walk
AppsFlyer:

```js
// In patch-rnfb-headers.js — refactor to walk multiple dirs:
const TARGET_DIRS = [
  '@react-native-firebase',
  'react-native-appsflyer',
];
for (const target of TARGET_DIRS) {
  const dir = path.join(__dirname, '..', 'node_modules', target);
  if (fs.existsSync(dir)) walk(dir);
}
```

Rename the file to `scripts/patch-native-modular-headers.js` since it's no longer
RNFB-specific. Update `package.json` postinstall reference + plugin name.

Build. Expected: PASS if AppsFlyer headers match the same `<React/X.h>` / `<RCTRequired/X.h>` / `<RCTTypeSafety/X.h>` patterns. If new patterns appear, extend regex like Phase 3.

### Option B — pivot to Adjust

[react-native-adjust](https://github.com/adjustcom/react-native-adjust) — modern, modular
headers from day 1, no RCT import bugs. Same use case (attribution for paid acquisition).

Replace `src/lib/appsflyer.ts` calls with Adjust SDK. Backend: log to Adjust events
endpoint. Migration time: ~2 hours.

Recommend Option A first (AppsFlyer is industry-standard, plus we already have credentials
provisioned in `_APPSFLYER/CREDENTIALS.md`). If Option A fails: Option B.
